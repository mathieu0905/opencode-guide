Plugin 插件系统

源码位置：packages/opencode/src/plugin/index.ts

概述

Plugin 系统让 OpenCode 可以通过外部代码扩展功能。插件可以提供自定义工具、 修改 System Prompt、拦截工具执行、注入环境变量、处理认证。 支持 NPM 包和本地文件两种加载方式。

Hook 详解 + 数据流

tool.execute.before / after — 工具拦截


  LLM 调用工具: bash({ command: "npm test" })
       │
       ▼
  ┌───────────────────────────────────────────────┐
  │  tool.execute.before                           │
  │                                               │
  │  input: {                                     │
  │    tool: "bash",                              │
  │    sessionID: "ses_001",                      │
  │    callID: "call_abc",                        │
  │    args: { command: "npm test" }              │
  │  }                                            │
  │                                               │
  │  插件可以:                                     │
  │    return null           → 继续正常执行        │
  │    return { output }     → 跳过执行, 直接返回  │
  │    修改 args             → 改变工具参数        │
  │    throw Error           → 拒绝执行            │
  └───────────────────┬───────────────────────────┘
                      │
                      ▼
            [工具实际执行]
                      │
                      ▼
  ┌───────────────────────────────────────────────┐
  │  tool.execute.after                            │
  │                                               │
  │  input: {                                     │
  │    tool: "bash",                              │
  │    args: { command: "npm test" },             │
  │    result: {                                  │
  │      title: "Run tests",                      │
  │      output: "5 passed, 0 failed",           │
  │      metadata: { exit: 0 }                    │
  │    }                                          │
  │  }                                            │
  │                                               │
  │  插件可以:                                     │
  │    return result          → 原样返回          │
  │    return modified_result → 修改输出          │
  │    追加日志/监控数据                            │
  └───────────────────────────────────────────────┘

  Bash 工具执行命令时:

      DATABASE_URL: "postgres://...",   // 注入数据库连接

  // 效果: spawn(command, { env: { ...process.env, ...pluginEnv } })

    "@my-org/my-plugin@^1.0.0",
    // → bun install 到项目的 .opencode/node_modules
    // → import("opencode-anthropic-auth")

    // ═══════ 方式 2: 本地文件 ═══════
    "file:///absolute/path/to/plugin.js",
    "file://./relative/path/plugin.ts",
    // → 直接 import()
    // → 支持 TypeScript (Bun 原生支持)
  ]
}

完整插件示例

// my-plugin.ts — 一个功能完整的插件

export default async function(input) {
  const { client, project, directory, $ } = input

  return {
    // ═══════ 监听事件 ═══════
    event({ event }) {
      if (event.type === "session.created") {
        console.log("New session:", event.properties.info.id)
      }
      if (event.type === "message.updated") {
        // 可以发送通知、写日志等
      }
    },

    // ═══════ 提供自定义工具 ═══════
    tool: {
      "deploy": {
        description: "Deploy the application",
        parameters: z.object({
          environment: z.enum(["staging", "production"]),
        }),
        execute: async (args, ctx) => {
          // 工具可以请求权限
          await ctx.ask({
            permission: "deploy",
            patterns: [args.environment],
          })

          // 执行部署
          const result = await $`deploy.sh ${args.environment}`.text()

          return {
            title: `Deploy to ${args.environment}`,
            metadata: { env: args.environment },
            output: result,
          }
        },
      },
    },

    // ═══════ 修改 System Prompt ═══════
    async "experimental.chat.system.transform"(input) {
      return {
        system: [
          ...input.system,
          `Project: ${project.name}`,
          "Always respond in Chinese.",
        ]
      }
    },

    // ═══════ 自定义认证 ═══════
    async auth(input) {
      if (input.provider === "my-private-llm") {
        const token = await fetchOAuthToken()
        return { api_key: token }
      }
    },

    // ═══════ 注入环境变量 ═══════
    async "shell.env"({ cwd }) {
      return {
        env: { PROJECT_ROOT: directory }
      }
    },
  }
}

  // .opencode/tools/deploy.ts
  import { z } from "zod"

  export default {
    id: "deploy",
    init: async () => ({
      description: "Deploy the application to production",
      parameters: z.object({
        environment: z.enum(["staging", "production"]),
      }),
      execute: async (args, ctx) => {
        await ctx.ask({
          permission: "deploy",
          patterns: [args.environment],
        })
        // ... 部署逻辑

          title: `Deploy to ${args.environment}`,
          metadata: {},
          output: "Deployment successful!",

      },

  }

  // ★ 工具会自动注册, LLM 可以直接调用 "deploy"
  // ★ 不需要 opencode.json 中的 plugin 配置

  │  自定义认证     │  ✕               │  ✓                    │

  建议:
    只需要自定义工具 → .opencode/tools/
    需要更多控制     → Plugin

设计要点

  • 异步初始化 — 插件函数是异步的，可以在初始化时做网络请求、 读取配置文件、建立数据库连接等。
  • Hook 链式执行 — 多个插件的同名 Hook 按注册顺序执行。 前一个插件的输出可以作为后一个的输入。
  • 两种扩展方式 — 简单场景用 .opencode/tools/（零配置）， 复杂场景用 Plugin（全部 Hook 能力）。
  • 受控上下文 — 插件通过 PluginInput 获取受控的上下文（SDK 客户端、 项目信息、Shell 访问），不能直接访问内部 API。
  • NPM 生态 — 插件可以发布到 NPM，版本号管理，bun install 自动安装依赖。
